# Feature Specification: 2048 遊戲

**Feature Branch**: `001-2048-game`  
**Created**: 2026-03-13  
**Status**: Draft  
**Input**: 做一個2048的遊戲  
**Language**: 本規格 MUST 使用繁體中文（專有名詞可保留英文）

## User Scenarios & Testing *(mandatory)*

> 憲章提醒：故事切分應以 MVP 為優先，避免過度設計。

### User Story 1 - 核心遊戲玩法 (Priority: P1)

玩家開啟網頁即可開始遊戲。遊戲板為 4×4 格子，初始出現兩個數字方塊（數值為 2 或 4）。玩家透過鍵盤方向鍵（或在觸控裝置上滑動）將所有方塊往同一方向移動，相鄰且數值相同的方塊會自動合併為兩倍數值。每次有效移動後，棋盤上的隨機空格會新增一個數值為 2 或 4 的方塊。玩家持續操作直到出現數值 2048 的方塊（獲勝）或棋盤填滿且無法移動（遊戲結束）。

**Why this priority**: 核心遊戲機制是整個功能的基礎，沒有它遊戲無法運作。此故事本身已是完整可玩的 MVP。

**Independent Test**: 可透過在空白棋盤上進行一系列滑動操作，驗證方塊移動、合併規則、新方塊出現，以及勝利／結束條件是否正確觸發。

**Acceptance Scenarios**:

1. **Given** 玩家開啟遊戲頁面，**When** 頁面載入完成，**Then** 顯示 4×4 空白棋盤並隨機出現 2 個初始方塊（數值為 2 或 4）
2. **Given** 棋盤有方塊且左側有空格，**When** 玩家按下左方向鍵，**Then** 所有方塊盡量向左滑動，且數值相同的相鄰方塊合併成兩倍數值
3. **Given** 玩家按下某方向鍵後棋盤有變化，**When** 移動完成，**Then** 在棋盤上隨機空格新增一個數值為 2（90% 機率）或 4（10% 機率）的方塊
4. **Given** 棋盤上出現數值 2048 的方塊，**When** 合併完成，**Then** 顯示「恭喜！你達到 2048！」的勝利提示，玩家可選擇繼續遊戲或重新開始
5. **Given** 棋盤已填滿且任意方向皆無法移動，**When** 玩家嘗試操作，**Then** 顯示「遊戲結束」提示，引導玩家重新開始

---

### User Story 2 - 分數記錄與最高分 (Priority: P2)

玩家在遊戲過程中可即時看到當前分數。每次方塊合併時，分數增加合併後方塊的數值（例如兩個 64 合併為 128，加 128 分）。頁面同時顯示本次遊玩期間的最高分記錄，讓玩家有持續挑戰自己的動力。

**Why this priority**: 分數系統讓遊戲更具可玩性與競爭感，但不影響核心遊戲機制，因此優先度次於 P1。

**Independent Test**: 可在已知棋盤狀態下執行合併操作，驗證分數計算是否正確累加，以及最高分是否在單次遊戲期間正確保留。

**Acceptance Scenarios**:

1. **Given** 玩家正在遊戲中，**When** 兩個數值相同的方塊合併，**Then** 當前分數即時增加合併後方塊的數值
2. **Given** 玩家重新開始遊戲，**When** 新遊戲開始，**Then** 當前分數歸零，但最高分保留本次遊玩期間的最高記錄
3. **Given** 玩家從未合併任何方塊，**When** 查看分數，**Then** 當前分數顯示為 0

---

### User Story 3 - 重新開始遊戲 (Priority: P3)

玩家在任何時候（遊戲進行中、遊戲結束或達到 2048 後）都可以透過「重新開始」按鈕快速重置遊戲。重置後棋盤清空並重新生成 2 個初始方塊，分數歸零。

**Why this priority**: 提供流暢的重玩體驗是必要的，但此功能依賴 P1 的核心機制，故為 P3。

**Independent Test**: 可在任意遊戲狀態下點擊「重新開始」，驗證棋盤是否清空、分數是否歸零、初始方塊是否正確出現。

**Acceptance Scenarios**:

1. **Given** 玩家在遊戲進行中，**When** 點擊「重新開始」按鈕，**Then** 棋盤清空、分數歸零，並重新出現 2 個初始方塊
2. **Given** 顯示遊戲結束畫面，**When** 玩家點擊「重新開始」，**Then** 遊戲正常重置並可繼續遊玩
3. **Given** 顯示勝利畫面，**When** 玩家點擊「重新開始」，**Then** 遊戲正常重置並可繼續遊玩

---

### Edge Cases

- 當玩家按下某方向鍵但棋盤上沒有任何方塊可移動時，棋盤保持不變且不新增方塊
- 同一行/列中有多對可合併的方塊時，合併從移動方向的最前端依序執行（一個方塊在同一次移動中只能合併一次）
- 棋盤僅剩一個空格時，新方塊仍應正確出現在該空格
- 遊戲結束的判斷需在所有合併完成後才執行，避免誤判
- 玩家在勝利提示出現後選擇「繼續遊戲」，遊戲應繼續運作且不再重複顯示勝利提示

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 遊戲 MUST 提供 4×4 格子的遊戲棋盤，供玩家進行滑動合併操作
- **FR-002**: 遊戲 MUST 支援上、下、左、右四個方向的方塊滑動操作
- **FR-003**: 遊戲 MUST 支援鍵盤方向鍵（↑↓←→）操作
- **FR-004**: 遊戲 MUST 支援觸控裝置的滑動手勢（上下左右 swipe）操作
- **FR-005**: 每次有效移動後，遊戲 MUST 在棋盤空格隨機生成一個數值為 2 或 4 的新方塊
- **FR-006**: 相同數值且相鄰（移動方向上相鄰）的方塊 MUST 在移動時自動合併，合併後數值為兩倍
- **FR-007**: 同一方塊在同一次移動中 MUST NOT 被合併超過一次
- **FR-008**: 遊戲 MUST 在棋盤出現數值 2048 的方塊時顯示勝利提示，並提供「繼續遊戲」與「重新開始」選項
- **FR-009**: 遊戲 MUST 在棋盤填滿且任意方向皆無法移動時顯示遊戲結束提示
- **FR-010**: 遊戲 MUST 即時顯示當前分數，每次合併時增加合併後方塊的數值
- **FR-011**: 遊戲 MUST 在頁面存活期間記錄並顯示最高分
- **FR-012**: 遊戲 MUST 提供「重新開始」按鈕，可在任何時候重置棋盤與當前分數
- **FR-013**: 遊戲開始時 MUST 隨機在棋盤上生成 2 個初始方塊
- **FR-014**: Implementation MUST preserve specification artifacts (`spec.md`, `plan.md`, `tasks.md`) and MUST NOT delete or overwrite them during scaffold/framework setup
- **FR-015**: 遊戲 MUST 採用靜態前端網頁架構，無需伺服器端處理

### Key Entities

- **遊戲棋盤（Board）**：4×4 的格子結構，儲存每格的方塊數值（0 代表空格），代表遊戲的當前狀態
- **方塊（Tile）**：棋盤上的數字單元，具有數值（2、4、8、16、32、64、128、256、512、1024、2048 等 2 的冪次），在移動時可滑動或合併
- **移動（Move）**：玩家輸入的一個方向指令，導致棋盤狀態改變（方塊滑動、合併、新方塊生成）
- **分數（Score）**：當前遊戲累計分數，初始為 0，每次合併增加合併後方塊的數值
- **最高分（Best Score）**：本次頁面存活期間的最高分記錄

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 玩家首次接觸遊戲後，在 1 分鐘內能理解規則並完成 5 次以上有效移動
- **SC-002**: 每次方塊移動操作的視覺回應時間不超過 100 毫秒，讓玩家感受流暢操作
- **SC-003**: 遊戲在所有主流現代瀏覽器（Chrome、Firefox、Safari、Edge 最新版）上正確運作，無功能性錯誤
- **SC-004**: 方塊合併邏輯完全符合原版 2048 規則，100% 通過標準合併規則測試案例
- **SC-005**: 遊戲支援桌面鍵盤與行動裝置觸控兩種操作方式，且兩種方式的操作成功率均達 100%
- **SC-006**: 遊戲結束與勝利條件在所有邊界情況下均能正確判斷，誤判率為 0%

## Assumptions

- 遊戲為單人離線遊玩，無需帳號或網路連線
- 最高分僅在頁面存活期間保留，關閉頁面後不持久化儲存（若需持久化，可於未來版本加入）
- 新方塊數值為 2（90% 機率）或 4（10% 機率），符合原版 2048 設計
- 遊戲棋盤固定為 4×4，不提供自訂大小功能（此為原版規格）
- 玩家達到 2048 後可選擇繼續遊戲，目標方塊可超過 2048（如 4096、8192 等），但勝利提示只出現一次
- 不需要動畫效果，但若效能允許，可加入方塊滑動視覺動畫以增強體驗
