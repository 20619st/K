# Data Model: 2048 遊戲

**Phase**: 1 — 設計  
**Branch**: `001-2048-game`  
**Date**: 2026-03-13

## 實體定義

### 1. Board（遊戲棋盤）

**描述**: 遊戲的核心狀態容器，持有 4×4 格子的當前數值。

```js
// 表示方式：number[][] (4 rows × 4 cols)
// 值域：0（空格）或 2 的正整數冪次（2, 4, 8, …, 2048, 4096, …）
let board = [
  [0, 0, 0, 0],
  [0, 0, 2, 0],
  [0, 4, 0, 0],
  [0, 0, 0, 0],
];
```

**欄位**:

| 欄位 | 類型 | 說明 |
|------|------|------|
| `board[r][c]` | `number` | 第 r 列第 c 欄的方塊數值；0 = 空格 |

**驗證規則**:
- 棋盤大小恆為 4×4，不可變更
- 值僅能為 0 或 2 的冪次（2^1 ~ 2^17 等）
- 空格數 ≥ 0，棋盤填滿時空格數 = 0

**狀態轉換**:
```
初始化 → [有 2 個隨機方塊]
→ 玩家輸入方向 → [執行 slide + merge]
→ 棋盤有變化？
    Yes → [生成新方塊] → 檢查勝利 → 檢查結束
    No  → [無動作，不生成新方塊]
```

---

### 2. Tile（方塊）

**描述**: 棋盤上具有數值的單元，以棋盤陣列中的整數表示。方塊無獨立物件，其
位置與數值均由棋盤陣列隱含表達。

| 屬性 | 類型 | 說明 |
|------|------|------|
| value | `number` | 方塊數值，2 的冪次（2, 4, … 2048, …） |
| row | `number` | 棋盤列索引（0–3） |
| col | `number` | 棋盤欄索引（0–3） |

> 實作注意：方塊不以獨立物件追蹤，而是透過棋盤陣列的位置索引隱含表達，以最簡化設計。

---

### 3. Move（移動）

**描述**: 玩家輸入的一個方向指令，觸發棋盤狀態變更。

| 屬性 | 類型 | 說明 |
|------|------|------|
| direction | `'up'｜'down'｜'left'｜'right'` | 移動方向 |

**處理流程**:
1. 旋轉棋盤使目標方向映射為「向左」
2. 對每一列執行 `slideRow(row)` → 回傳新列與得分增量
3. 旋轉回原方向
4. 比較新舊棋盤：若有變化，生成新方塊

**`slideRow(row: number[]): { result: number[], delta: number }`**:
1. 過濾零值（壓縮非空方塊）
2. 從左到右依序合併相鄰相同值（每個方塊只合併一次，FR-007）
3. 補零至長度 4
4. 回傳結果列與得分增量（合併後數值之和）

---

### 4. Score（分數）

**描述**: 當前遊戲累計分數。

| 屬性 | 類型 | 說明 |
|------|------|------|
| score | `number` | 當前分數，初始為 0 |
| bestScore | `number` | 本次頁面存活期間最高分，初始為 0 |

**驗證規則**:
- `score ≥ 0`（不可為負）
- 每次合併後：`score += mergedTileValue`
- 每次移動後：`bestScore = Math.max(bestScore, score)`
- 重新開始：`score = 0`（`bestScore` 保留）

---

### 5. GameState（遊戲狀態）

**描述**: 完整遊戲狀態的容器。

```js
const state = {
  board: number[][],      // 4×4 棋盤
  score: number,          // 當前分數
  bestScore: number,      // 最高分
  status: 'playing' | 'won' | 'over', // 遊戲狀態
  hasWon: boolean,        // 是否已顯示過勝利提示（確保只顯示一次）
};
```

**狀態機**:

```
playing ──合併出 2048 且 !hasWon──→ won (hasWon = true)
won     ──繼續遊戲──→ playing
playing ──棋盤滿且無合法移動──→ over
over    ──重新開始──→ playing
won     ──重新開始──→ playing (hasWon = false)
playing ──重新開始──→ playing (hasWon = false)
```

## 關鍵演算法摘要

### slideRow

```
Input:  [2, 0, 2, 4]
Step 1 (壓縮): [2, 2, 4]
Step 2 (合併): [4, 4]  → delta += 4
Step 3 (補零): [4, 4, 0, 0]
Output: { result: [4, 4, 0, 0], delta: 4 }
```

### isGameOver

```
若 board 有空格 → false
否則，對每個格子，檢查其右鄰與下鄰是否數值相同
若有任一相同 → false（仍有合法移動）
否則 → true
```

### spawnTile

```
隨機選取一個 board[r][c] === 0 的位置
value = Math.random() < 0.9 ? 2 : 4
```
